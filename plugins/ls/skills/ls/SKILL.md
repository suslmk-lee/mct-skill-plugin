# ls (Linear Skill)

Linear 이슈 관리와 Git 브랜치/PR 워크플로우를 통합합니다.

## 트리거 조건

다음 상황에서 자동 적용됩니다:

- `{TEAM-KEY}-{숫자}` 패턴 언급 시 (예: ADE-24, BYU-12)
- "이슈 시작", "작업 시작", "브랜치 만들어" 발화 시 → `/ls:start` 실행
- "PR 만들어", "PR 올려", "풀리퀘" 발화 시 → `/ls:pr` 실행
- "이슈 완료", "done", "닫아", "마무리" 발화 시 → `/ls:done` 실행
- "지금 뭐 하고 있어", "현재 작업", "무슨 이슈" 발화 시 → `/ls:status` 실행
- "이슈 목록", "할 일 목록", "이슈 뭐 있어" 발화 시 → `/ls:list` 실행
- "서브이슈 통합", "integrate", "통합 PR", "sub-issue PR" 발화 시 → `/ls:integrate` 실행

## 세션 컨텍스트 유지 규칙

- `/ls:start` 실행 후 출력된 **현재 작업 세션** 블록을 대화 전반에 걸쳐 기억한다
- 이후 코드 작성·리뷰·PR 생성 시 이슈의 수락기준을 참조한다
- 수락기준 달성 여부를 판단할 수 있으면 능동적으로 사용자에게 보고한다
- `/ls:pr` 실행 시 세션 컨텍스트의 수락기준이 충족됐는지 PR 본문에 체크리스트로 포함한다

## API 호출 규칙

- 모든 GraphQL 요청은 인라인 curl로 처리한다 (외부 스크립트 import 금지)
- 한글 포함 텍스트: `curl -d` 인라인 금지 → 임시 파일 생성 후 `--data-binary @tmpfile` 사용
- 임시 파일은 항상 `/tmp/linear-XXXXXX.json` 패턴으로 생성하고 작업 완료 즉시 삭제한다
- API 에러 응답 `{"errors":[...]}` 패턴 감지 시 에러 내용을 사용자에게 알린다
- Slack webhook 실패는 무시하고 계속 진행한다 (비핵심 기능)

## 이슈 키 처리 규칙

- 이슈 식별자(예: ADE-24) → 팀 키 `ade` + 번호 `24`로 분리해 GraphQL filter로 UUID 조회
- 현재 브랜치명에서 이슈 키 자동 추출: `ade-24-xxx` → `ADE-24`
- 추출 실패 시 사용자에게 이슈 키 직접 요청
